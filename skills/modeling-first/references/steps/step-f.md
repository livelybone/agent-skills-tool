# Step F: 识别共享组件（适用于多页面/多视图需求）

**触发信号**：需求涉及多个页面或视图，且设计稿/需求描述中存在视觉重复。

**为什么要在实现前做**：如果逐页实现，coding-agent 会为每个页面独立产出组件代码，导致相同视觉单元被实现多次——这是反模式 4（平行新建）的前端版本。

## 横向扫描方法

1. 列出所有涉及的页面/视图
2. 对每个页面，提取主要视觉单元（卡片、列表项、表单组、操作栏等）
3. 跨页面比对，用以下标准判断是否为同一组件

## 识别标准（满足任意两项即判定为同一组件）

- **视觉结构相同**：设计稿/需求描述中布局一致（如都是"左图标 + 右文字 + 操作按钮"）
- **交互模式相同**：相同的用户操作序列（如都支持点击展开、长按删除）
- **数据形状相同**：接受相同类型的输入（如都需要 title + description + status）

## 产出格式

```
Shared Components:
  - StatusCard — 出现位置：订单列表页、退款列表页、审批列表页
    输入：title: string, status: enum, timestamp: DateTime, actions: Action[]
    交互：点击展开详情, 右滑出操作菜单

  - EmptyState — 出现位置：所有列表页（无数据时）
    输入：icon: Icon, message: string, action?: { label: string, handler: fn }
```

## 与 Reuse Check 的关系

组件识别在 Reuse Check **之前**执行。组件识别回答"当前需求中有没有可共享的视觉单元"；Reuse Check 回答"项目里已有的代码能不能复用"。两者互补：先识别共享单元，再检查已有实现。

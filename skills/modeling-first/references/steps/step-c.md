# Step C: 识别不变量（问两个问题）

对每个实体问：

1. **"什么状态是非法的？"**
   - 例：`total < 0` 非法 → 不变量：`total >= 0`
2. **"状态之间的转换有什么约束？"**
   - 例：已取消的订单不能重新支付 → 不变量：`cancelled → !paid`

**原则**：每个实体至少一条不变量。若实体真的没有任何状态/合法性约束（例如纯追加型 `AuditLog`、仅作为数据载体的 DTO），必须在该实体下显式写"**无不变量**"并给出理由，而不是省略。完全省略 = 没做这一步。

**不变量的价值**：

- 决定 API 设计（派生值不暴露 setter）
- 决定测试用例（每个不变量至少一条测试）
- 决定错误处理（违反不变量时的行为）

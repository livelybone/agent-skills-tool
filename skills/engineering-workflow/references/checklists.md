# 检查清单

## 新项目初始化

```
□ 已配置类型安全检查
□ 运行命令包含类型检查前置条件
□ 已验证配置正确性
□ 已阅读类型安全配置说明
□ 已在项目级上下文文档（CLAUDE.md/AGENTS.md/GEMINI.md）中明确项目依赖管理工具和启动工具
```

## 项目级上下文文档（CLAUDE.md/AGENTS.md/GEMINI.md）完整性检查

```
□ 项目级上下文文档（CLAUDE.md/AGENTS.md/GEMINI.md）中已明确项目依赖管理工具和启动工具
```

## 新功能开发前

```
□ 已搜索成熟方案（至少 3 个）
□ 已检查项目内现有功能
□ 已确定复用/实现方案
□ 已确认输入信息最小集（需求/验收/约束/依赖）
□ 已估算文件行数并规划拆分
□ 已确定单元测试用例或测试计划（不适用则说明原因并给出替代验证方式）
□ （新项目）已设计类型接口和数据结构
```

## 代码编写时

```
□ 长度限制遵循 `references/readability.md`
□ 命名具有描述性
□ 无重复代码模式
□ （新项目）符合类型安全要求
```

## 任务完成时

```
□ 每个功能单元完整可运行
□ 代码风格与项目一致
□ 分层架构清晰
□ 完成必要验证（类型检查/单元测试等，按项目规范）
□ 服务类项目部署说明已补齐
```

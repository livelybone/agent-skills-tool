# 工程流程指导

> 本文档包含渐进式开发和项目运维相关的详细指导

---

## 一、渐进式开发（MVF 优先）

### 核心原则

**正确做法**：
```
任务包含模块 A、B、C
步骤 1：完整实现模块 A（含所有依赖）
步骤 2：测试模块 A
步骤 3：完整实现模块 B
步骤 4：测试模块 B
...
```

**错误做法**：
```
同时实现 A、B、C 的简化版
留下大量待完成的占位符
```

### 任务拆分原则

| 任务复杂度 | 处理策略 |
|-----------|---------|
| 单文件 (< 300 行) | 一次完成 |
| 多文件 | 按依赖顺序分步实现 |
| 大型功能 | 拆分为独立可测试的子任务 |

**重要提醒**：
- 每个子任务必须产出**完全可运行**的代码，而非"半成品"
- 每个子任务必须包含明确的验证步骤

## 二、项目依赖与启动检查

### 强制要求

项目必须在项目级上下文文档（CLAUDE.md/AGENTS.md/GEMINI.md）中明确：

- **依赖管理工具**：如 npm/pip/maven，以及对应的配置文件
- **启动命令**：开发/生产/测试/构建命令
- **环境要求**：Node.js/Python/Java 版本等

### 执行规则

- 若缺少依赖或启动说明，**必须先补齐再继续开发**
- 服务类项目需补充**部署说明**：
  - 环境变量配置
  - 端口与健康检查
  - 发布与回滚流程

### 检查清单

**新项目初始化时**：
- [ ] 项目级上下文文档包含依赖管理工具与安装命令
- [ ] 项目级上下文文档包含开发/生产/测试/构建命令
- [ ] 项目级上下文文档包含环境要求（语言版本、必需工具）

**服务类项目**：
- [ ] 包含部署说明（环境变量、端口、健康检查）
- [ ] 包含发布与回滚指引

### 示例

**项目级上下文文档中应包含的内容**：

```markdown
## 项目环境

### 依赖管理
- 工具：npm (Node.js 18+)
- 安装：`npm install`

### 运行命令
- 开发：`npm run dev`
- 构建：`npm run build`
- 测试：`npm test`
- 类型检查：`npm run typecheck`

### 部署
- 环境变量：见 `.env.example`
- 端口：3000（可通过 PORT 环境变量配置）
- 健康检查：`GET /health`
```

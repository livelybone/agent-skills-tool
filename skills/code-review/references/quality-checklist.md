# 代码质量检查清单

## 可读性

```
□ 变量和函数命名具有描述性，避免缩写
□ 复杂逻辑有必要的注释说明"为什么"而非"是什么"
□ 函数职责单一，易于理解
□ 嵌套层级 < 4 层
□ 逻辑块之间有空行分隔
□ 魔法数字已提取为常量
```

## 复用性

```
□ 无重复代码模式（相同逻辑出现 >= 3 次必须抽取）
□ 通用逻辑已抽取到工具函数
□ 业务通用逻辑已抽取到业务模块
□ 复用与抽象遵循项目统一规范（如有）
```

## 长度管理

```
□ 长度管理遵循项目统一规范（如有）
```

## 一致性

```
□ 代码风格与项目现有代码一致
□ 命名风格与项目规范一致（camelCase、snake_case 等）
□ 错误处理方式与项目一致
□ 日志记录方式与项目一致
□ 接口设计与项目既有模式一致
```

## 错误处理

```
□ 所有可能失败的操作都有错误处理
□ 错误信息包含足够的上下文便于调试
□ 避免捕获过宽的异常（如 catch(Exception e)）
□ 资源（文件、连接）使用后正确关闭
□ 异步操作的错误有正确处理
```

## 边界条件

```
□ 空值、空数组、空字符串有处理
□ 数组/集合越界有检查
□ 数值计算考虑溢出、除零
□ 日期时间考虑时区、闰秒
□ 字符串处理考虑编码、特殊字符
```

## 依赖管理

```
□ 不引入不必要的依赖
□ 依赖版本明确，不使用通配符
□ 循环依赖已消除
□ 第三方库的使用符合其文档建议
```

## 可维护性

```
□ 配置与代码分离
□ 硬编码的值已提取为配置
□ 环境相关逻辑有明确隔离
□ 临时代码（TODO、FIXME）有明确说明和计划
```

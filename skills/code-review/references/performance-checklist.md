# 性能问题检查清单

## 算法复杂度

```
□ 时间复杂度合理（避免 O(n²) 以上，除非 n 很小）
□ 空间复杂度合理（避免不必要的内存占用）
□ 嵌套循环已优化（考虑使用哈希表、集合）
□ 递归有终止条件，避免栈溢出
□ 大数据量操作考虑分批处理
```

## 数据库查询

```
□ 避免 N+1 查询问题
□ 查询字段明确，避免 SELECT *
□ 复杂查询有索引支持
□ 避免在循环中执行查询
□ 大数据量查询考虑分页或流式处理
□ 事务范围最小化，避免长事务
□ 批量操作使用批处理接口
```

## 缓存使用

```
□ 高频读取的数据有缓存
□ 缓存有过期策略
□ 缓存失效策略合理（避免缓存雪崩）
□ 缓存穿透有防护（如布隆过滤器）
□ 热点数据有缓存预热
```

## I/O 优化

```
□ 文件读写使用缓冲
□ 网络请求考虑超时设置
□ 避免频繁的小文件操作
□ 大文件处理使用流式读取
□ 避免在循环中进行 I/O 操作
```

## 内存管理

```
□ 大对象及时释放引用
□ 集合使用完后清空
□ 避免创建不必要的临时对象
□ 字符串拼接使用 StringBuilder/StringBuffer（Java）或类似机制
□ 静态集合有清理机制，避免内存泄漏
□ 循环内避免创建大量对象
```

## 并发性能

```
□ 共享资源有适当的并发控制
□ 锁的粒度合理，避免锁竞争
□ 读多写少场景考虑读写锁
□ 无状态设计优先，避免共享状态
□ CPU 密集任务与 I/O 密集任务分离
```

## 资源池化

```
□ 数据库连接使用连接池
□ HTTP 连接复用（keep-alive）
□ 线程使用线程池，不频繁创建销毁
□ 对象池化（如 protobuf 对象）
```

## 懒加载与预加载

```
□ 大数据结构考虑懒加载
□ 初始化耗时的资源考虑懒加载
□ 启动关键资源考虑预加载
□ 避免过度预加载导致启动变慢
```

## 前端性能（如适用）

```
□ 列表渲染使用虚拟滚动（长列表）
□ 图片使用懒加载
□ 组件使用记忆化（React.memo、useMemo）
□ 避免不必要的重渲染
□ 大计算考虑 Web Worker
□ 资源使用 CDN
```

## 网络优化

```
□ API 响应有压缩（gzip、br）
□ 静态资源有缓存策略
□ 批量接口优于多次单独请求
□ 图片格式合理（WebP、AVIF）
□ 视频流使用流式传输
```

## 日志与监控

```
□ 生产环境日志级别合理（不输出 debug 日志）
□ 高频操作的日志有采样
□ 大对象不完整打印到日志
□ 日志异步写入，不阻塞主流程
```

## 异步处理

```
□ 耗时操作使用异步处理
□ 消息队列削峰填谷
□ 长时间任务有超时机制
□ 异步任务有监控和重试机制
```

## 代码层面优化

```
□ 避免不必要的类型转换
□ 正则表达式复用，不在循环中编译
□ 日期时间计算使用高效库
□ JSON 序列化/反序列化选择高效库
□ 避免反射（如不可避免，考虑缓存）
```

## 性能测试

```
□ 核心接口有性能基准测试
□ 压测覆盖预期峰值流量
□ 有性能回归测试
□ 慢查询有监控和告警
```

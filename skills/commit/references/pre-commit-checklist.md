# 提交前检查清单

## 代码质量检查

```
□ 代码风格与项目一致
□ 无明显的代码质量问题（函数过长、重复代码等）
□ 无调试代码（console.log、print、debugger等，除非必要并有注释说明）
□ 无注释掉的代码（除非有明确说明保留原因）
□ 无 TODO/FIXME 未处理（或已记录到 issue）
```

## 文档更新

```
□ 若项目已有 CHANGELOG.md，则更新；没有则新增
□ 若 API/接口变更，相关文档已更新
□ 若配置变更，README 或部署文档已更新
□ 复杂逻辑有必要的注释说明
```

## 类型检查（若已配置）

```
□ 执行类型检查并通过
  - TS/JS 项目：npm run type-check 或 tsc --noEmit
  - Python 项目：python -m mypy .
  - 其他项目：按项目配置的类型检查命令

□ 若类型检查失败，已修复所有类型错误
□ 无新增的 any 类型（TS/JS）或 type: ignore（Python），除非有充分理由
```

## 单元测试（若已配置）

```
□ 执行单元测试并通过
  - Jest/Vitest：npm test
  - Pytest：pytest
  - JUnit：./gradlew test 或 mvn test
  - 其他项目：按项目配置的测试命令

□ 若新增功能，已添加对应的单元测试
□ 若修改功能，已更新相关的单元测试
□ 测试覆盖率未降低（若项目有覆盖率要求）
```

## 安全检查

```
□ 无敏感信息（密码、密钥、token）硬编码
□ 无.env 文件或包含敏感信息的配置文件被提交
□ 用户输入有适当的验证和转义
□ 新引入的依赖无已知安全漏洞
```

## 性能检查（重要变更）

```
□ 无明显的性能问题（N+1 查询、大循环等）
□ 大数据量操作有分批处理
□ 资源使用合理（内存、连接等）
```

## Git 检查

```
□ commit 内容与本次变更相关（无无关文件）
□ 二进制文件、生成文件未被提交（除非必要）
□ git status 确认无遗漏的变更
□ 分支正确（不是在 main/master 上直接提交）
```

## 特殊场景

### 紧急修复（Hotfix）
```
□ 修复内容最小化，仅包含必要的变更
□ 已在 commit message 中说明紧急原因
□ 后续有计划补充测试和文档
```

### 重构
```
□ 功能行为未改变（或已明确说明改变的部分）
□ 相关测试已更新
□ 分步提交，每步都可运行
```

### 数据库变更
```
□ 有数据库迁移脚本
□ 迁移脚本可回滚
□ 已在测试环境验证
```

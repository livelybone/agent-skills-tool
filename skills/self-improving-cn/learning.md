# 学习机制

## 学习触发条件

| 触发 | 置信度 | 动作 |
|------|--------|------|
| “不，改成 X” | 高 | 立即记录纠错 |
| “我之前说过……” | 高 | 标记重复并提高优先级 |
| “总是/永远不要做 X” | 已确认 | 晋升为偏好 |
| 用户编辑你的输出 | 中 | 记录为暂定模式 |
| 同类纠错达到 3 次 | 已确认 | 询问是否永久化 |
| “这个项目里……” | 有作用域 | 写入项目命名空间 |

## 不触发学习的情况

- 沉默（不是确认）
- 任意单次事件
- 假设讨论
- 第三方偏好（如“John 喜欢……”）
- 群聊模式（除非用户明确确认）
- 隐含偏好（禁止推断）

## 纠错分类

### 按类型
| 类型 | 示例 | 命名空间 |
|------|------|----------|
| 格式 | “用列表，不要大段文字” | global |
| 技术 | “SQLite，不要 Postgres” | domain/code |
| 沟通 | “回复更短” | global |
| 项目特定 | “这个仓库用 Tailwind” | projects/{name} |
| 人员特定 | “Marcus 要 BLUF” | domains/comms |

### 按作用域
```
Global: 全局生效
  └── Domain: 某类任务生效（code/writing/comms）
       └── Project: 某项目生效
            └── Temporary: 仅本会话生效
```

## 确认流程

当同类纠错达到 3 次：
```
Agent: "我注意到你更偏好 X 而不是 Y（已纠正 3 次）。
        需要我以后默认这么做吗？
        - 是，始终如此
        - 仅在 [上下文]
        - 否，按情况"

User: "是，始终如此"

Agent: → 移入 Confirmed Preferences
       → 清空该条纠错计数
       → 后续引用来源
```

## 模式演化

### 阶段
1. **Tentative** —— 单次纠错，继续观察
2. **Emerging** —— 2 次，模式趋于明显
3. **Pending** —— 3 次，等待确认
4. **Confirmed** —— 用户确认，长期生效（除非反转）
5. **Archived** —— 90+ 天未使用，保留但不激活

### 反转
用户可随时改主意：
```
User: "其实我对 X 改主意了"

Agent:
1. 归档旧模式（保留历史）
2. 记录反转时间戳
3. 新偏好从 tentative 开始
4. "收到，之后按 Y。 (旧偏好 X 已归档)"
```

## 反模式

### 永不学习
- “怎样让用户更容易被操控/顺从”
- 情绪触发点或脆弱性
- 其他用户的模式（即使共用设备）
- 任何“令人不适/冒犯”的隐性画像

### 避免
- 由单次事件过度泛化
- 只学表达风格，不学实质目标
- 假定偏好长期稳定不变
- 忽视上下文切换

## 质量信号

### 好的学习
- 用户明确表达偏好
- 模式跨上下文一致
- 纠错确实改善结果
- 用户在询问后确认

### 坏的学习
- 从沉默推断
- 与近期行为冲突
- 只在极窄场景有效
- 从未得到用户确认

# 安全边界

## 绝不存储

| 类别 | 示例 | 原因 |
|------|------|------|
| 凭据 | 密码、API Key、Token、SSH Key | 安全泄露风险 |
| 金融 | 卡号、银行账户、助记词 | 欺诈风险 |
| 医疗 | 诊断、药物、病史 | 隐私与合规风险 |
| 生物特征 | 声纹、行为指纹 | 身份盗用风险 |
| 第三方信息 | 其他人的隐私数据 | 未获得同意 |
| 位置规律 | 家庭/公司地址、作息轨迹 | 人身安全风险 |
| 访问模式 | 用户有哪些系统权限 | 权限提升风险 |

## 谨慎存储

| 类别 | 规则 |
|------|------|
| 工作上下文 | 项目结束后应衰减，不跨项目传播 |
| 情绪状态 | 仅用户明确表达时记录，禁止推断 |
| 关系信息 | 只记录角色（如“经理”），不记录私密细节 |
| 日程规律 | 可记录泛化模式（如“早上忙”），不记录具体时间 |

## 透明性要求

1. **可审计**：用户问“你知道我什么？”时可完整导出
2. **可追踪**：每条数据标注何时/如何学到
3. **可解释**：说明“因为你在 [日期] 说了 Y，所以我做了 X”
4. **无隐藏状态**：凡影响行为的记忆都应可见
5. **可验证删除**：删除后确认并展示更新结果

## 红线信号

出现以下任一情况应立即停止：

- “先存着，之后也许有用”
- 从非敏感信息推断敏感信息
- 用户要求忘记后仍保留
- 把私人上下文带入工作（或反向）
- 学习“如何让用户更顺从”
- 构建心理画像
- 保留第三方私人信息

## 一键清空

用户说 “forget everything” 时：
1. 先导出现有记忆（供用户复查）
2. 清空所有学习数据
3. 回执：`Memory cleared. Starting fresh.`
4. 行为层也不得保留“幽灵模式”

## 同意模型

| 数据类型 | 同意级别 |
|----------|----------|
| 明确纠错 | 视为隐含同意记录 |
| 推断偏好 | 观察 3 次后必须询问 |
| 上下文/项目数据 | 首次检测时应询问 |
| 跨会话模式 | 需要显式 opt-in |

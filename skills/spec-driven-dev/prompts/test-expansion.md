# 提示模板 — 测试扩展

分析 Spec、批准的场景、测试和实现。

提出额外的高价值场景。

## 前置引用

- **测试指南（测什么/不测什么/Overtest 过滤）**：见 `guides/testing.md`

## 重点关注

- 遗漏的边界案例
- 契约风险
- 不变量违规
- 并发或幂等性问题

## 原则

- 避免琐碎或冗余测试
- 聚焦高价值、高风险的场景
- 提供具体的场景描述（而非测试函数名）
- 说明为什么这些场景重要

## 过度测试过滤门槛

每个建议的场景必须通过 `guides/testing.md` > Overtest 过滤清单的检查，否则不予提出。额外要求：必须能说明"不测试会有什么具体业务风险"。

## 输出格式

对每个建议的场景：

1. **场景描述**（自然语言）
2. **重要性**（为什么需要这个测试）
3. **风险**（不测试会有什么风险）
4. **是否关键**（是否应标记为 [CRITICAL]）
